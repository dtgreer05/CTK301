Lecture Video (40 minutes)

Lecture Notes

Links

<p><a href="index.html">This is the text for the link.</a> This is text outside the link.</p>
  • <a> - surrounds one or more words to create a link
  • href - the place the link goes to
  • relative (internal) links - other pages that are on your web site (you just need the filename)
  • Links (<a> tags) are inline elements (they don't break the flow of text)
  • absolute (external) links - pages elsewhere on the web (don't forget the https://)
<p>Welcome to <a href="https://www.illinoisstate.edu" target="_blank">Illinois State University</a>!</p>
  • target="_blank" - opens the link in a new tab

Navigation

  • Use an unordered list of links to connect the pages of your site
<ul>
  <li><a href="basic.html">Home</a></li>
  <li><a href="continued.html">Basic HTML, Continued</a></li>
  <li><a href="page3.html">Page 3</a></li>
</ul>

Images

<img src="image.jpg" alt="Image Description" >
  • <img> - it's a "singular" tag, no closing tag needed!
  • src - the filename of the image
  • alt - alternate text for the image
    • Always put an alt on images!
    • Displays if image is broken
    • Required for accessibility
  • Image file formats
    • JPEG - good for real world images
    • GIF - good for images with solid blocks of color (logos)
    • PNG - like GIF but better, transparency but no animation Links to an external site.
    • JPG vs. PNG? - check size and quality when exporting from Photoshop
    • RGB - web images should be RGB- CMYK might not work in browser
    • TIFF, HEIC - won't work in browser!
  • Troubleshooting
    • When uploading your page online, make sure you upload the linked images along with your page!
    • If your image works in a page when you look at it locally (Open in Browser) but it doesn't work online. Check the filename! picture.jpg is not the same as Picture.JPG on the web server!

Folders and Paths

CTK301
  basic.html
  continued.html
  myimage1.jpg
  ↳ folder
    subpage.html
    myimage2.jpg
  • The src in img tags and the href in a tags are "paths" or directions to the files.
  • If the file or image is in the same folder as the page, you can just use the file name in the src or href.
  • If a file or image is in a folder below, you'll need to add the folder name and "/" to the path. 
    <!-- continued.html -->

<img src="folder/myimage2.jpg" alt="Image Description" >

<a href="folder/subpage.html">A link to a page in a folder below</a>
  • If a file or image is in a folder above, you'll need to add "../" to the path. 
    <!-- subpage.html -->

<img src="../myimage1.jpg" alt="Image Description" >

<a href="../continued.html">A link to a page in a folder above</a>

Deprecations and Proper Usage

  • <center> - Don't use! Deprecated, use styles instead
  • <u> - underline, deprecated but back in HTML5 but not a good idea anyways
  • <table> - OK to use for tabular data but don't use for page structure!
